import ConfigGenerator from '../../../components/config-generator/ConfigGenerator'

export const kaliVariantList = [
  "标准（deb + deb-src）",
  "仅二进制（deb）",
];

export const defaultVersion = "标准（deb + deb-src）";

export const GenKaliConfig = (variant) => {
  if (variant === "仅二进制（deb）") {
    return `deb https://mirrors.jcut.edu.cn/kali kali-rolling main non-free contrib`;
  }
  return `deb https://mirrors.jcut.edu.cn/kali kali-rolling main non-free contrib
deb-src https://mirrors.jcut.edu.cn/kali kali-rolling main non-free contrib`;
};

## Kali 镜像使用帮助

### 地址

[https://mirrors.jcut.edu.cn/kali/](https://mirrors.jcut.edu.cn/kali/)

### 说明

*Kali Linux* 软件源

### 支持的系统架构

amd64, armel, armhf, i386

---

### 使用说明

#### 方法一：直接编辑 sources.list

选择所需的源类型，复制后**覆盖写入** `/etc/apt/sources.list`，或添加到文件最顶部：

<ConfigGenerator promptString="请选择源类型：" versionList={kaliVariantList} defaultVersion={defaultVersion} configGen={GenKaliConfig} language="bash" />

> 如无需自行编译软件包，选择「仅二进制」即可，`deb-src` 行对日常使用无影响。

复制后执行：

```bash
sudo tee /etc/apt/sources.list << EOF
# 粘贴上方内容
EOF
sudo apt-get update
```

#### 方法二：一键 sed 替换

如果您的 `/etc/apt/sources.list` 中已有官方源，可直接替换地址：

```shell
sudo sed -i "s@http://http.kali.org/kali@https://mirrors.jcut.edu.cn/kali@g" /etc/apt/sources.list
```

---

### 注意

> 更改完 `sources.list` 文件后请运行 `sudo apt-get update` 更新索引以生效。

由于 Kali 仓库未使用 by-hash 机制，同步时的一致性无法保证。如果在同步时执行 `apt update`，可能会看到以下错误

```text
E: 无法下载 https://mirrors.jcut.edu.cn/kali/dists/kali-rolling/main/source/Sources.gz 文件尺寸不符(14593053 != 14592993)。您使用的镜像正在同步中？
```

如果出现以上错误，则需等待同步完成后再执行 `apt update`。
