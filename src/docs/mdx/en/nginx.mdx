### Installation Instructions

Before installing nginx on a new machine for the first time, you need to set up the nginx package repository. After that, you can install and update nginx from the repository.

#### RHEL and Derivatives

This section applies to Red Hat Enterprise Linux and its derivatives, such as CentOS, Oracle Linux, Rocky Linux, and AlmaLinux.

Install prerequisites:

```bash
sudo yum install yum-utils
```

To set up the yum repository, create a file named `/etc/yum.repos.d/nginx.repo` with the following content:

```text
[nginx-stable]
name=nginx stable repo
baseurl=http://mirrors.jcut.edu.cn/nginx/centos/$releasever/$basearch/
gpgcheck=1
enabled=1
gpgkey=https://nginx.org/keys/nginx_signing.key
module_hotfixes=true

[nginx-mainline]
name=nginx mainline repo
baseurl=http://mirrors.jcut.edu.cn/nginx/mainline/centos/$releasever/$basearch/
gpgcheck=1
enabled=0
gpgkey=https://nginx.org/keys/nginx_signing.key
module_hotfixes=true
```

By default, the stable nginx repository is used. If you want to use the mainline nginx packages, run the following command:

```bash
sudo yum-config-manager --enable nginx-mainline
```

To install nginx, run the following command:

```bash
sudo yum install nginx
```

When prompted to accept the GPG key, verify that the fingerprint matches `573B FD6B 3D8F BC64 1079 A6AB ABF5 BD82 7BD9 BF62`, and if so, accept it.

#### Ubuntu

Install prerequisites:

```bash
sudo apt install curl gnupg2 ca-certificates lsb-release ubuntu-keyring
```

Import the official nginx signing key so that `apt` can verify the authenticity of the packages. Get the key:

```bash
curl https://nginx.org/keys/nginx_signing.key | gpg --dearmor \
| sudo tee /usr/share/keyrings/nginx-archive-keyring.gpg >/dev/null
```

Verify that the downloaded file contains the correct key:

```bash
gpg --dry-run --quiet --no-keyring --import --import-options import-show /usr/share/keyrings/nginx-archive-keyring.gpg
```

The output should contain the full fingerprint `573BFD6B3D8FBC641079A6ABABF5BD827BD9BF62`, as shown below:

```text
pub   rsa2048 2011-08-19 [SC] [expires: 2027-05-24]
    573BFD6B3D8FBC641079A6ABABF5BD827BD9BF62
uid                      nginx signing key <signing-key@nginx.com>
```

Note that the output may contain other keys used to sign packages.

To set up the apt repository to use stable nginx packages, run the following command:

```bash
echo "deb [signed-by=/usr/share/keyrings/nginx-archive-keyring.gpg] \
http://mirrors.jcut.edu.cn/nginx/ubuntu `lsb_release -cs` nginx" \
| sudo tee /etc/apt/sources.list.d/nginx.list
```

If you want to use mainline nginx packages, run the following command:

```bash
echo "deb [signed-by=/usr/share/keyrings/nginx-archive-keyring.gpg] \
http://mirrors.jcut.edu.cn/nginx/mainline/ubuntu `lsb_release -cs` nginx" \
| sudo tee /etc/apt/sources.list.d/nginx.list
```

Set up repository pinning to prefer our packages over those provided by the distribution:

```bash
echo -e "Package: *\nPin: origin nginx.org\nPin: release o=nginx\nPin-Priority: 900\n" \
| sudo tee /etc/apt/preferences.d/99nginx
```

To install `nginx`, run the following command:

```bash
sudo apt update
sudo apt install nginx
```

#### SLES

Install prerequisites:

```bash
sudo zypper install curl ca-certificates gpg2
```

To set up the zypper repository to use stable nginx packages, run the following command:

```bash
sudo zypper addrepo --gpgcheck --type yum --refresh --check \
    'http://mirrors.jcut.edu.cn/nginx/sles/$releasever_major' nginx-stable
```

If you want to use mainline nginx packages, run the following command:

```bash
sudo zypper addrepo --gpgcheck --type yum --refresh --check \
    'http://mirrors.jcut.edu.cn/nginx/mainline/sles/$releasever_major' nginx-mainline
```

Next, import the official nginx signing key so that zypper/rpm can verify the authenticity of the packages. Get the key:

```bash
curl -o /tmp/nginx_signing.key https://nginx.org/keys/nginx_signing.key
```

Verify that the downloaded file contains the correct key:

```bash
gpg --with-fingerprint /tmp/nginx_signing.key
```

The output should contain the full fingerprint `573B FD6B 3D8F BC64 1079 A6AB ABF5 BD82 7BD9 BF62`, as shown below:

```text
pub  2048R/7BD9BF62 2011-08-19 [expires: 2027-05-24]
       Key fingerprint = 573B FD6B 3D8F BC64 1079 A6AB ABF5 BD82 7BD9 BF62
uid nginx signing key <signing-key@nginx.com>
```

Finally, import the key into the rpm database:

```bash
sudo rpmkeys --import /tmp/nginx_signing.key
```

To install nginx, run the following command:

```bash
sudo zypper install nginx
```

#### Alpine

Install prerequisites:

```bash
sudo apk add openssl curl ca-certificates
```

To set up the apk repository to use stable nginx packages, run the following command:

```bash
printf "%s%s%s%s\n" \
    "@nginx " \
    "http://mirrors.jcut.edu.cn/nginx/alpine/v" \
    `egrep -o '^[0-9]+\.[0-9]+' /etc/alpine-release` \
    "/main" \
    | sudo tee -a /etc/apk/repositories
```

If you want to use mainline nginx packages, run the following command:

```bash
printf "%s%s%s%s\n" \
    "@nginx " \
    "http://mirrors.jcut.edu.cn/nginx/mainline/alpine/v" \
    `egrep -o '^[0-9]+\.[0-9]+' /etc/alpine-release` \
    "/main" \
    | sudo tee -a /etc/apk/repositories
```

Next, import the official nginx signing key so that apk can verify the authenticity of the packages. Get the key:

```bash
curl -o /tmp/nginx_signing.rsa.pub https://nginx.org/keys/nginx_signing.rsa.pub
```

```bash
openssl rsa -pubin -in /tmp/nginx_signing.rsa.pub -text -noout

```

```bash

sudo yum install yum-utils
```
